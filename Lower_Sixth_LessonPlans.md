Lower Sixth — 30‑Week Ready-to-Teach Lesson Plans

Template for each lesson:

- Duration: 40–60 minutes
- Learning Objectives: 2–4 measurable outcomes
- Materials & Setup
- Prior Knowledge
- Starter (5–10 min)
- Teaching Activities (20–30 min)
- Student Practice (15–20 min)
- Assessment / Exit Ticket
- Homework / Extension
- Differentiation
- Resources

Week 1 — Introduction to Computer Science

- Duration: 50 min
- Learning Objectives: Define computer science; list career paths; explain course structure and assessment.
- Materials: syllabus handout, projector, whiteboard
- Prior Knowledge: none required
- Starter: discuss devices students use daily; quick brainstorm: "What is CS?"
- Teaching: present course overview, exam expectations, assessment rubric; discuss classroom norms and safety.
- Practice: students map 3 goals and sign learning contract.
- Assessment: exit ticket — 3 things learned and one question.
- Homework: read syllabus and prepare one career-related question.
- Differentiation: provide glossary for unfamiliar terms.
- Resources: course syllabus, sample career profiles.

Week 2 — History & Scope of Computing; Ethics

- Duration: 50 min
- Learning Objectives: Summarize computing history; discuss ethics and digital citizenship.
- Materials: slides, short article on computing milestones
- Starter: timeline activity: place milestones in order (group)
- Teaching: lecture on milestones, evolution of hardware/software; discuss privacy and ethics.
- Practice: small groups debate a scenario (privacy vs convenience).
- Assessment: exit ticket — define digital citizenship.
- Homework: write 200-word reflection on ethical use of data.
- Differentiation: provide sentence starters for reflections.
- Resources: selected readings, short video links.

Week 3 — Number Systems: Binary & Hex

- Duration: 50 min
- Learning Objectives: Convert between decimal, binary and hexadecimal; explain bit/byte.
- Materials: conversion worksheet, calculators, projector
- Starter: mental quick quiz converting small numbers.
- Teaching: demonstrate place values, conversions (decimal→binary; binary→hex).
- Practice: paired conversion exercises, teacher circulates.
- Assessment: exit ticket — convert 45 to binary.
- Homework: set of 10 conversions.
- Differentiation: scaffold with conversion steps; extension: two's complement intro.
- Resources: worksheets, conversion tool links.

Week 4 — Character Encoding & Data Sizes

- Duration: 50 min
- Learning Objectives: Explain ASCII and Unicode; calculate storage for strings.
- Materials: sample texts, worksheet
- Starter: ask how many characters in their full name; estimate storage.
- Teaching: explain ASCII limits, Unicode basics, bytes per character, multi-byte encodings.
- Practice: convert short text to ASCII codes; calculate storage for given strings.
- Assessment: exit ticket — difference between ASCII and Unicode.
- Homework: compare storage of same text in ASCII and UTF-8.
- Differentiation: offer worked examples; extension: UTF-16 discussion.
- Resources: encoding charts.

Week 5 — Boolean Algebra: Basics

- Duration: 50 min
- Learning Objectives: Use AND/OR/NOT; construct truth tables.
- Materials: truth table worksheet, logic gate images
- Starter: short puzzles using true/false statements.
- Teaching: define AND/OR/NOT, truth tables, boolean expressions.
- Practice: students complete truth tables and simplify expressions.
- Assessment: exit ticket — truth table for (A AND NOT B).
- Homework: boolean simplification problems.
- Differentiation: pair weaker students with peers; extension: introduce XOR.
- Resources: worksheets, interactive simulators.

Week 6 — Logic Gates & Circuits (Intro)

- Duration: 50 min
- Learning Objectives: Identify basic gates; draw simple combinational circuits.
- Materials: circuit diagrams, online simulator (if available)
- Starter: identify logic gate symbols.
- Teaching: map boolean expressions to gate diagrams; build simple circuits (adder conceptually).
- Practice: translate expressions ↔ diagrams.
- Assessment: exit ticket — draw gate for A OR (NOT B).
- Homework: small worksheet mapping expressions to gates.
- Differentiation: provide templates for drawing circuits.
- Resources: online logic simulators.

Week 7 — Combinational Logic Examples (Adders, Multiplexers)

- Duration: 50 min
- Learning Objectives: Explain half/full adder; multiplexer function.
- Materials: diagrams, truth tables
- Starter: recap gates and truth tables.
- Teaching: show half-adder truth table and gate layout; explain multiplexer/select lines.
- Practice: students complete truth tables and sketch simple mux.
- Assessment: exit ticket — state purpose of multiplexer.
- Homework: problems on adder truth tables.
- Differentiation: guided worksheet for beginners.
- Resources: slides and examples.

Week 8 — Computer Architecture: CPU, Memory, I/O

- Duration: 60 min
- Learning Objectives: Describe CPU components, memory hierarchy, I/O basics.
- Materials: diagram handouts
- Starter: "What happens when you press power?" quick brainstorm.
- Teaching: discuss ALU, control unit, registers, RAM vs storage.
- Practice: label a block diagram; match components to functions.
- Assessment: exit ticket — role of registers.
- Homework: short report on a common CPU component.
- Differentiation: extra reading for high achievers.
- Resources: diagrams, short explainer videos.

Week 9 — Operating Systems: Roles & Processes

- Duration: 50 min
- Learning Objectives: Explain OS responsibilities, processes, and file systems.
- Materials: case studies, slides
- Starter: list tasks your OS performs.
- Teaching: cover process scheduling, file management, drivers.
- Practice: scenario-based questions (what does OS do?).
- Assessment: exit ticket — define a process.
- Homework: compare two OS features (e.g., Windows vs Linux basics).
- Differentiation: scaffolded notes for novices.
- Resources: readings, diagrams.

Week 10 — Problem Solving & Pseudocode

- Duration: 50 min
- Learning Objectives: Apply stepwise refinement; write pseudocode for simple problems.
- Materials: problem set, flowchart paper
- Starter: everyday algorithm discussion (making tea).
- Teaching: show pseudocode conventions and flowchart symbols.
- Practice: students write pseudocode for finding max of three numbers.
- Assessment: exit ticket — one benefit of pseudocode.
- Homework: pseudocode for grading system.
- Differentiation: provide skeletons for beginners.
- Resources: sample pseudocode.

Week 11 — Programming Fundamentals: Variables & Types

- Duration: 60 min
- Learning Objectives: Declare variables; use integers, floats, strings; basic I/O.
- Materials: classroom IDE or REPL, starter code
- Starter: quick quiz on variable names and legality.
- Teaching: live-code variable declaration and printing, type conversion.
- Practice: students write program to compute area of rectangle from input.
- Assessment: teacher inspects code; exit ticket asks to explain a type error.
- Homework: small program reading name and age.
- Differentiation: scaffolded starter code; extension: format output.
- Resources: language reference sheets.

Week 12 — Expressions & Operators

- Duration: 50 min
- Learning Objectives: Use arithmetic and logical operators; precedence rules.
- Materials: exercises, IDE
- Starter: mental operator precedence quiz.
- Teaching: demonstrate expressions and operator precedence, parentheses use.
- Practice: evaluate expressions and debug wrong outputs.
- Assessment: exit ticket — evaluate given expression.
- Homework: worksheet on operator precedence.
- Differentiation: extra challenge problems for advanced students.
- Resources: operator table.

Week 13 — Control Structures: Selection

- Duration: 50 min
- Learning Objectives: Implement if/else constructs to control program flow.
- Materials: IDE, code examples
- Starter: decision-making scenarios (group)
- Teaching: show if, if-else, elif structures; nested conditions.
- Practice: students write program to grade scores into categories.
- Assessment: exit ticket — write condition for pass/fail.
- Homework: program with multiple conditions.
- Differentiation: provide test cases for novices.
- Resources: sample solutions.

Week 14 — Control Structures: Iteration

- Duration: 60 min
- Learning Objectives: Use for and while loops; avoid infinite loops.
- Materials: IDE, loop exercises
- Starter: predict outputs of simple loops.
- Teaching: iterate over ranges, lists; loop control statements (break/continue).
- Practice: implement summation and factorial using loops.
- Assessment: exit ticket — write loop to print even numbers to 20.
- Homework: iterative Fibonacci generator.
- Differentiation: pair students for peer support.
- Resources: examples and hints.

Week 15 — Procedural Decomposition: Functions

- Duration: 60 min
- Learning Objectives: Define functions, use parameters/return values, understand scope.
- Materials: IDE, function worksheet
- Starter: refactor a repeated code block discussion.
- Teaching: live-code function creation and calls, scope examples.
- Practice: create functions for unit conversion.
- Assessment: exit ticket — advantages of functions.
- Homework: write reusable functions for common tasks.
- Differentiation: provide templates for beginners.
- Resources: sample function library.

Week 16 — Data Structures: Arrays & Lists

- Duration: 60 min
- Learning Objectives: Use arrays/lists to store and manipulate collections.
- Materials: IDE, sample arrays
- Starter: small problem: store student marks and compute average.
- Teaching: indexing, traversal, basic operations (append, pop).
- Practice: implement simple list-based tasks (search, sum).
- Assessment: exit ticket — difference between array and individual variables.
- Homework: list-based exercise set.
- Differentiation: extension: nested lists.
- Resources: examples.

Week 17 — File Handling Basics

- Duration: 50 min
- Learning Objectives: Read and write text files; understand file modes.
- Materials: IDE, sample text files
- Starter: discuss uses of files vs variables.
- Teaching: open/read/write/close; error handling basics.
- Practice: write program to count words in a file.
- Assessment: exit ticket — sample code line to open a file for reading.
- Homework: create program to append user entries to a file.
- Differentiation: provide file templates.
- Resources: file API docs.

Week 18 — Sorting & Searching (Intro)

- Duration: 60 min
- Learning Objectives: Implement bubble, selection, insertion sorts; perform linear and binary search.
- Materials: sample datasets, IDE
- Starter: ask how to find a name in an unsorted list.
- Teaching: demonstrate bubble and selection sort, explain binary search prerequisites.
- Practice: students code bubble sort and test on small lists.
- Assessment: exit ticket — explain why binary search is faster on sorted data.
- Homework: implement binary search.
- Differentiation: provide step-by-step algorithm traces for novices.
- Resources: pseudocode handouts.

Week 19 — Recursion (Basics)

- Duration: 60 min
- Learning Objectives: Understand recursion and write simple recursive functions (factorial, Fibonacci).
- Materials: recursion worksheets, IDE
- Starter: compare iterative vs recursive solutions for factorial.
- Teaching: base case, recursive call, stack visualization.
- Practice: implement factorial and recursive sum; trace calls.
- Assessment: exit ticket — what is base case and why needed.
- Homework: recursive implementation of power function.
- Differentiation: visual aids and recursion trace tools.
- Resources: step-through examples.

Week 20 — Introduction to Object-Oriented Concepts

- Duration: 60 min
- Learning Objectives: Define class, object, attribute and method; create simple class.
- Materials: IDE, example class code
- Starter: everyday objects mapping to classes.
- Teaching: class syntax, constructor, instance methods.
- Practice: implement a `Student` class with methods to compute average.
- Assessment: exit ticket — instance vs class attribute.
- Homework: extend class with validation.
- Differentiation: skeleton code for beginners.
- Resources: OOP reference.

Week 21 — Databases Fundamentals: Tables & SQL (Intro)

- Duration: 60 min
- Learning Objectives: Describe table structure; run basic SELECT/INSERT/UPDATE/DELETE.
- Materials: SQLite or sample DB, sample queries
- Starter: how would you store student records on disk?
- Teaching: relational tables, primary keys, basic SQL statements.
- Practice: run queries against sample DB and modify records.
- Assessment: exit ticket — simple SELECT query.
- Homework: create table and insert 5 records.
- Differentiation: guided SQL scripts.
- Resources: SQLite docs, sample DB files.

Week 22 — Networks Fundamentals & Internet Basics

- Duration: 50 min
- Learning Objectives: Define LAN/WAN; explain client-server and basic protocols (HTTP/TCP/IP).
- Materials: network diagrams, slides
- Starter: map devices in school network.
- Teaching: explain packet switching, IP addresses, DNS.
- Practice: simple exercises mapping addresses and explaining role of DNS.
- Assessment: exit ticket — what does HTTP do?
- Homework: research how emails are routed (brief).
- Differentiation: reading list for deeper understanding.
- Resources: diagrams and videos.

Week 23 — Web Basics: HTML & CSS

- Duration: 60 min
- Learning Objectives: Create a basic HTML page and style with CSS.
- Materials: text editor, browser, template files
- Starter: show a simple webpage and discuss structure.
- Teaching: HTML tags, basic CSS selectors, inline vs external styles.
- Practice: students create a personal profile page.
- Assessment: exit ticket — list three HTML tags and their purpose.
- Homework: improve page with CSS layout.
- Differentiation: provide template; extension: responsive basics.
- Resources: MDN intro pages.

Week 24 — Spreadsheets & Productivity Tools

- Duration: 50 min
- Learning Objectives: Use formulas, create charts, basic macros.
- Materials: spreadsheet software, sample data
- Starter: show gradebook example.
- Teaching: formulas, cell referencing, charts, simple macro recording.
- Practice: compute averages and create a pie chart of grades.
- Assessment: exit ticket — formula for average.
- Homework: build a budget sheet.
- Differentiation: templates for novices.
- Resources: spreadsheet cheat sheet.

Week 25 — Software Development Lifecycle & Testing

- Duration: 50 min
- Learning Objectives: Describe SDLC stages; write basic test cases.
- Materials: slides, sample test cases
- Starter: discuss past projects and planning steps.
- Teaching: phases from requirements to maintenance, unit testing concepts.
- Practice: write test cases for small functions and run tests.
- Assessment: exit ticket — one benefit of testing.
- Homework: write two unit tests for homework code.
- Differentiation: sample tests provided.
- Resources: testing framework docs.

Week 26 — Computer Security & Digital Citizenship

- Duration: 50 min
- Learning Objectives: Identify common threats; demonstrate safe practices.
- Materials: case studies, slides
- Starter: discuss recent cyber incidents (local/global).
- Teaching: passwords, phishing, malware basics, responsible conduct online.
- Practice: analyze scenarios and propose safe responses.
- Assessment: exit ticket — list 3 password best practices.
- Homework: create a security checklist for home devices.
- Differentiation: scaffolded checklist template.
- Resources: cybersecurity primer.

Week 27 — Practical Labs: Integrated Exercises

- Duration: 60–90 min (double period recommended)
- Learning Objectives: Integrate programming, file I/O, and simple data processing.
- Materials: IDE, datasets, lab instructions
- Starter: recap recent skills.
- Teaching: brief demo of lab tasks and expectations.
- Practice: students complete lab exercises in pairs.
- Assessment: lab submission and teacher feedback.
- Homework: finish any incomplete lab tasks.
- Differentiation: tiered tasks within lab.
- Resources: lab instructions and datasets.

Week 28 — Mini Project Development: Planning

- Duration: 60 min
- Learning Objectives: Produce project proposal, break down tasks, set milestones.
- Materials: project brief template, rubric
- Starter: present mini-project examples.
- Teaching: explain deliverables and marking criteria.
- Practice: students draft proposals and receive teacher feedback.
- Assessment: submit proposal for approval.
- Homework: finalize proposal and start implementation.
- Differentiation: provide example proposals.
- Resources: template and rubric.

Week 29 — Mini Project Implementation

- Duration: 60–90 min
- Learning Objectives: Implement project features; apply testing and version control basics.
- Materials: repo access, lab machines
- Starter: checklist review.
- Teaching: quick refresher on git basics and testing expectations.
- Practice: students code and commit progress; teacher circulates.
- Assessment: commit history and working demo.
- Homework: complete project and prepare presentation.
- Differentiation: scaffold tasks by difficulty.
- Resources: sample codebase, checklist.

Week 30 — Mini Project Presentation & Assessment

- Duration: 60–90 min
- Learning Objectives: Present project, demonstrate functionality, reflect on learning.
- Materials: presentation rubric, demo setup
- Starter: expectations for presentations.
- Teaching: students present for 5–10 minutes each; peer feedback.
- Practice: Q&A and reflections.
- Assessment: rubric-based grading (functionality, code, documentation, presentation).
- Homework: final report submission.
- Differentiation: allow written presentations where speaking is a barrier.
- Resources: grading rubric.

End of Lower Sixth plans (30 weeks).
