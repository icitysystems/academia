Upper Sixth — 30‑Week Ready-to-Teach Lesson Plans

Template for each lesson:

- Duration: 40–90 minutes (some topics may require double periods)
- Learning Objectives: 2–4 measurable outcomes
- Materials & Setup
- Prior Knowledge
- Starter (5–10 min)
- Teaching Activities (20–40 min)
- Student Practice (20–30 min)
- Assessment / Exit Ticket
- Homework / Extension
- Differentiation
- Resources

Week 1 — Algorithm Analysis & Big O

- Duration: 60 min
- Learning Objectives: Explain Big O notation; compute time/space complexity for simple algorithms.
- Materials: slides, problem sets
- Starter: compare two algorithms by steps count.
- Teaching: introduce Big O, examples (linear, quadratic, logarithmic), best/worst/average case.
- Practice: analyze provided pseudocode.
- Assessment: exit ticket — classify complexity of sample code.
- Homework: complexity exercises.
- Differentiation: worked examples for novices.
- Resources: complexity tables.

Week 2 — Stacks & Queues

- Duration: 60 min
- Learning Objectives: Implement stack/queue operations and apply to problems.
- Materials: IDE, visual aids
- Starter: real-world analogies (plates, queues).
- Teaching: LIFO vs FIFO, push/pop/enqueue/dequeue.
- Practice: implement stack & queue; use for simple expression evaluation.
- Assessment: exit ticket — when to use a queue.
- Homework: bracket matching using a stack.
- Differentiation: starter code provided.
- Resources: examples.

Week 3 — Linked Lists

- Duration: 60 min
- Learning Objectives: Implement singly linked list insertion, deletion, traversal.
- Materials: IDE, pointer diagrams
- Starter: discuss dynamic collection vs static arrays.
- Teaching: node structure, head pointer, operations.
- Practice: implement and test list methods.
- Assessment: exit ticket — complexity of insert at head vs tail.
- Homework: reverse a linked list.
- Differentiation: visual trace for novices.
- Resources: sample code.

Week 4 — Trees: Terminology & Traversals

- Duration: 60 min
- Learning Objectives: Define tree, binary tree; implement inorder/preorder/postorder traversals.
- Materials: diagrams, IDE
- Starter: family tree analogy.
- Teaching: node, root, leaf, height, traversals (recursive).
- Practice: code traversals and test.
- Assessment: exit ticket — traversal orders for sample tree.
- Homework: build tree from given list.
- Differentiation: traversal trace worksheets.
- Resources: visualization tools.

Week 5 — Binary Search Trees

- Duration: 60 min
- Learning Objectives: Insert/search in BST; explain balanced vs unbalanced trees.
- Materials: IDE, sample datasets
- Starter: sorting vs search discussion.
- Teaching: BST properties, insertion algorithm.
- Practice: implement BST insert and search; test edge cases.
- Assessment: exit ticket — difference between BST and binary tree.
- Homework: BST deletion (beginner version).
- Differentiation: guided steps for insertion.
- Resources: BST visualizers.

Week 6 — Heaps & Priority Queues

- Duration: 60 min
- Learning Objectives: Explain heap structure; use heap for priority queue and heap sort concept.
- Materials: diagrams, heap operations worksheet
- Starter: priority scenarios (hospital triage).
- Teaching: array representation, sift-up/sift-down.
- Practice: implement basic heap operations.
- Assessment: exit ticket — heapify steps for small list.
- Homework: use heap to sort small array.
- Differentiation: step-by-step help for novices.
- Resources: heap animation tools.

Week 7 — Hash Tables & Collision Strategies

- Duration: 60 min
- Learning Objectives: Design hash functions; implement chaining/open addressing basics.
- Materials: IDE, worksheets
- Starter: map keys to buckets analogy.
- Teaching: hash functions, collisions, load factor, chaining vs probing.
- Practice: implement simple hash table with chaining.
- Assessment: exit ticket — explain load factor effect.
- Homework: design a simple hash function for strings.
- Differentiation: provide starter hash functions.
- Resources: examples.

Week 8 — Merge Sort & Quick Sort

- Duration: 60–90 min
- Learning Objectives: Implement merge sort and quicksort; analyze complexities.
- Materials: pseudocode handouts, IDE
- Starter: compare simple sorts vs advanced sorts.
- Teaching: divide-and-conquer paradigm; demonstrate merge and partition steps.
- Practice: implement merge sort and quicksort; test and compare timings on sample data.
- Assessment: exit ticket — explain where quicksort performs poorly.
- Homework: analyze recursion depth for merge sort.
- Differentiation: scaffold merge procedure.
- Resources: visualizers and pseudocode.

Week 9 — Dynamic Programming (Intro)

- Duration: 60 min
- Learning Objectives: Identify overlapping subproblems; apply memoization to Fibonacci and knapsack (small).
- Materials: DP worksheets, IDE
- Starter: compare naive vs memoized Fibonacci times.
- Teaching: explain memoization and bottom-up DP.
- Practice: refactor recursive Fibonacci to memoized version.
- Assessment: exit ticket — explain memoization benefit.
- Homework: implement DP solution for coin change (small case).
- Differentiation: stepwise hints.
- Resources: DP examples.

Week 10 — Advanced Recursion & Backtracking (Intro)

- Duration: 60 min
- Learning Objectives: Solve small backtracking problems (permutations, simple maze).
- Materials: IDE, worksheet
- Starter: permutations of letters quick activity.
- Teaching: recursion with branching and pruning.
- Practice: implement permutations generator.
- Assessment: exit ticket — base case explanation for permutations.
- Homework: implement simple backtracking search.
- Differentiation: provide recursion trees for novices.
- Resources: examples.

Week 11 — Advanced OOP: Inheritance & Polymorphism

- Duration: 60 min
- Learning Objectives: Apply inheritance and polymorphism in designs; override methods safely.
- Materials: UML diagrams, IDE
- Starter: role-based examples (vehicles -> car/truck)
- Teaching: class hierarchies, super calls, interfaces/abstract classes.
- Practice: implement class hierarchy and polymorphic behavior.
- Assessment: exit ticket — advantage of polymorphism.
- Homework: design an inheritance model for school staff.
- Differentiation: provide UML templates.
- Resources: OOP reference.

Week 12 — Design Patterns (Intro)

- Duration: 60 min
- Learning Objectives: Recognize and apply simple patterns: factory, singleton, observer.
- Materials: pattern cheat sheet, IDE
- Starter: real-world factory analogy.
- Teaching: explain intent and example code for each pattern.
- Practice: implement a simple factory pattern in code.
- Assessment: exit ticket — when to use singleton.
- Homework: refactor small code to use a design pattern.
- Differentiation: guided refactor tasks.
- Resources: pattern references.

Week 13 — Version Control with Git (Practical)

- Duration: 60 min
- Learning Objectives: Initialize repo, stage/commit, branch and merge basics.
- Materials: Git installed or online GUI (GitHub classroom)
- Starter: discuss code collaboration issues without VCS.
- Teaching: git basics, branching model and commit messages.
- Practice: students create repo, commit changes, push to remote.
- Assessment: exit ticket — sequence of commands to create a branch.
- Homework: create a small project repository with README.
- Differentiation: GUI steps for beginners.
- Resources: git reference commands.

- Duration: 60 min
- Learning Objectives: Write unit tests; understand CI pipeline basics.
- Materials: testing framework, sample tests
- Starter: what happens when code breaks in production?
- Teaching: unit test concepts, assertions, test cases, simple CI demo.
- Practice: write tests for existing functions and run locally.
- Assessment: exit ticket — why unit tests matter.
- Homework: write tests for homework project code.
- Differentiation: starter tests provided.
- Resources: testing docs.

- Duration: 60 min
- Learning Objectives: Normalize schema to 3NF; write joins for queries.
- Materials: sample ER diagrams, SQL environment
- Starter: discuss data redundancy examples.
- Teaching: normalization rules, join types, transactions basics.
- Practice: normalize provided schema and write join queries.

Week 16 — Advanced SQL & Indexing

- Duration: 60 min
- Learning Objectives: Use complex queries, understand indexing and query plans.
- Materials: sample DB, explain plan examples
- Starter: slow query example discussion.
- Teaching: multi-table joins, group by, indexing basics and when to use indexes.
- Practice: optimize sample query with index suggestions.

Week 17 — RESTful APIs & Backend Intro

- Duration: 60–90 min

- Materials: IDE, lightweight framework (Flask/Express)
- Starter: discuss how web apps talk to servers.
- Teaching: endpoints, routing, JSON payloads, status codes.

- Homework: add validation and error handling.
- Differentiation: starter repo for novices.
- Resources: API docs.

- Materials: browser, code examples
- Starter: show fetch example and demo.

- Practice: build small client that lists resources from Week 17 API.

- Learning Objectives: Create project plan, milestones, testing strategy, and repo structure.
- Materials: project rubric, project management template

Week 30 — Capstone Implementation & Presentation

- Duration: 90+ min (finals & presentations)
- Learning Objectives: Complete capstone deliverables; present and defend work; reflect on learning.
- Materials: presentation rubric, demo environment
- Starter: expectations and presentation schedule.

- Differentiation: allow alternative deliverables or recorded demos if necessary.

End of Upper Sixth plans (30 weeks).
